# Frontend Prototype Analysis

> Source: `docs/IGFEJ/prototypes/frontend-prototype`

## Purpose

The frontend prototype demonstrates how IGFEJ process information can be explored through a public-facing catalog with hierarchical navigation, process detail pages, subprocess detail pages, and BPMN visualisation.

It is not a production application. It is a reference prototype used to validate:

- information architecture
- navigation flows
- process/subprocess page layouts
- BPMN viewing behavior
- search, filtering, sorting, and pagination

## Current Functional Scope

### Static sections

- `#welcome`
- `#dashboard`
- `#introduction`
- `#methodology`
- `#faq`

### Dynamic routes

- `#macroprocess/{id}`
- `#process/{id}/{tab}`
- `#subprocess/{id}/{tab}`
- `#asset/{id}`

### Process detail tabs

- `Visão Geral`
- `Subprocessos`
- `Diagramas`
- `Versões`

### Subprocess detail tabs

- `Detalhes`
- `Diagrama`

## Key Behaviors

### 1. Hierarchical navigation

Users can navigate:

- from macroprocesses to processes
- from processes to subprocesses
- from processes and subprocesses to diagrams/assets

Breadcrumbs and menu state are synchronized with the current route.

### 2. Refresh-safe routing

The prototype uses hash routing and restores the current page when the browser refreshes.

Examples:

- `#dashboard`
- `#process/c6cb6f1f-96a9-4b63-85a8-827a63b8f76f/diagrams`
- `#subprocess/1b6aa1c4-7f40-4c3e-bbb8-6c6d4f4be0d5/diagram`

### 3. Search

There are two global searches:

- welcome-page search
- expandable header search

Current search behavior includes:

- accent-insensitive matching
- code matching such as `5.1`
- tokenized queries such as `5.1: criacao`
- highlighted matches
- result navigation to process, subprocess, or asset pages

### 4. Dashboard tables

The dashboard exposes two shared table components:

- `Todos os Macroprocessos`
- `Todos os Processos`

Current table features:

- text search
- process filters
- sortable columns
- pagination
- fixed-height rows to reduce pagination jumping

### 5. BPMN viewer

The BPMN experience is shared between process and subprocess pages.

Supported actions:

- fit to viewport
- zoom in/out
- fullscreen
- download available formats only

The viewer uses shared logic so process and subprocess diagram tabs behave consistently.

## Information Architecture Assessment

### Strengths

- Clear hierarchy between macroprocess, process, subprocess, and diagram
- Good separation between process summary pages and detailed subprocess/process content
- Search works across multiple entity types
- Diagram viewing is integrated into the navigation flow
- Hash routing makes deep-linking easy during analysis sessions

### Issues that were addressed in the prototype evolution

- duplicated titles and metadata in process overviews
- inconsistent tab naming such as `Histórico` vs `Versões`
- process/subprocess landing layouts using different visual models
- duplicated BPMN viewer wrapper code
- unstable search focus/close behavior
- anchor navigation lag
- dashboard table toolbar inconsistency
- table pagination/footer movement caused by varying row heights

### Current layout direction

The current prototype now favors:

- summary-first overview tabs for process pages
- direct-entry subprocess detail pages with header summaries
- lighter page hierarchy
- fewer nested cards
- shared table behavior
- shared diagram viewer behavior

## Technical Structure

The frontend is split by responsibility:

### `core/`

- bootstrap
- state
- routing
- layout
- shared view switching

### `components/`

- reusable page fragments
- shared tables
- shared process/subprocess rendering
- shared diagram viewer shell

### `sections/`

- top-level menu-backed pages

### `routes/`

- dynamic detail-page controllers

### `utils/`

- BPMN handling
- search
- small visual helpers

This structure reduced duplication and made the prototype easier to change without keeping all logic in one file.

## Data Model Used by the Prototype

### `macroprocesses.json`

Provides:

- macroprocess metadata
- descriptions
- top-level counts

### `processes.json`

Provides:

- process metadata
- versions
- subprocesses
- diagram assets
- BPMN/SVG/DMN references

### `search-index.json`

Provides:

- prebuilt global search entries
- process, subprocess, and asset records

## Prototype Limitations

- mock-data driven, not backed by a real API
- no authentication or authorization
- no persistence of edits
- no CMS workflow
- no real-time analytics source
- some BPMN assets are reused as representative sample diagrams rather than one-to-one final mappings

## Conclusion

The prototype is already strong enough to validate the public consultation experience for process data.

Its biggest value is in showing:

- how information should be grouped
- how users move between hierarchy levels
- how diagrams fit into the browsing flow
- how search and dashboard discovery support exploration
